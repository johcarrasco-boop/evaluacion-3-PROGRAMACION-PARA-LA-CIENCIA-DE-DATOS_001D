# Evidencia sugerida de trabajo colaborativo en Git

Para cumplir la rúbrica, registrar en GitHub:

## Ramas recomendadas
- `main`: versión estable.
- `develop`: integración general.
- `feature/etl-pipeline`: desarrollo ETL.
- `feature/dashboard-streamlit`: dashboard.
- `feature/api-rest`: API FastAPI.
- `feature/docker-deploy`: Docker y despliegue.
- `feature/docs`: documentación.

## Commits sugeridos
- `feat(etl): integrar fuentes CSV SQLite y API`
- `feat(dashboard): agregar vistas ejecutiva tecnica y operativa`
- `feat(api): exponer endpoints de metricas de churn`
- `test(etl): agregar validaciones automatizadas`
- `docs: agregar arquitectura y guia de despliegue`
- `build(docker): agregar docker compose para api y dashboard`

## Issues sugeridas
1. Crear pipeline ETL robusto.
2. Diseñar dashboard por audiencia.
3. Documentar arquitectura y despliegue.
4. Agregar pruebas automatizadas.
5. Containerizar servicios.

## Pull requests sugeridos
- PR 1: ETL base + validaciones.
- PR 2: API REST.
- PR 3: Dashboard.
- PR 4: Docker + documentación.
